# bvm-lang v0.2.5 — Crate Rename

**A rename, not a rewrite.** The crate name `vm-lang` was already registered on crates.io by another owner, so this project publishes as **`bvm-lang`** and its library imports as **`bvm_lang`**. There is no functional change from `0.2.0`: the public API, execution semantics, and behavior are identical. Only the name moved.

## What changed

- The package is now `bvm-lang` (was `vm-lang`).
- The library is now `bvm_lang` (was `vm_lang`). Imports become `use bvm_lang::...`.
- Crate metadata, README, `docs/API.md`, the CHANGELOG, and the examples were updated to the new name.

Nothing else was touched — `Op`, `Chunk`, `Vm`, `VmError`, the instruction set, the numeric semantics, and the `serde` feature are exactly as shipped in `0.2.0`.

## Migration

If you referenced the crate under its development name, change the dependency and the import:

```toml
[dependencies]
bvm-lang = "0.2.5"
```

```rust
// before: use vm_lang::{Chunk, Op, Vm};
use bvm_lang::{Chunk, Op, Vm};
```

## Breaking changes

**None** at the API level. The crate and library identifiers changed, which is the entire purpose of this release; code using the types is otherwise unaffected.

## Verification

Re-ran the full gate after the rename on Windows x86_64 (Rust stable 1.95) and Linux (WSL2 Ubuntu, Rust stable 1.96); MSRV 1.85 verified locally:

```bash
cargo fmt --all -- --check
cargo clippy --all-targets --all-features -- -D warnings
cargo test --all-features
cargo test --no-default-features
RUSTDOCFLAGS="-D warnings" cargo doc --no-deps --all-features
cargo deny check
```

All green. Test counts are unchanged from `0.2.0`: 40 unit + 11 integration (incl. 3 `serde`) + 7 property + 8 doctests under `--all-features`.
