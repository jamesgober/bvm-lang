# bvm-lang v1.0.0 — API Freeze

**The public API is now stable and frozen.** From this release, `bvm-lang` follows Semantic Versioning with a firm promise: no breaking change to the documented surface until a `2.0.0`. A pre-freeze adversarial review of the API caught two things worth fixing while they still could be — both are the breaking changes this major bump carries. Everything else is unchanged from `0.2.5`.

## What is bvm-lang?

A small, fast register-based bytecode virtual machine — the execution engine an interpreted language runs on once its source has been compiled to instructions. You assemble a `Chunk` of `Op` instructions over a constant pool and run it with a `Vm`, which returns a `Value` (the eight-byte NaN-boxed type from `value-lang`). Instructions are fixed-size decoded values dispatched through a single `match` loop; the crate is safe Rust (`unsafe` is forbidden) and treats bytecode as untrusted input — a malformed program returns a typed error, never a panic.

## The freeze

[`docs/STABILITY.md`](../STABILITY.md) is the new source of truth. It records:

- The exact frozen surface: `Vm`, `Chunk`, `Op`, `VmError`, the `Reg`/`Const`/`Addr` aliases, and the re-exported `Value`/`Unpacked`/`Symbol`.
- What may still change **additively** within `1.x` without breaking: new `Op` variants and new `VmError` variants (both enums are `#[non_exhaustive]`), new inherent methods, and new types — for example an execution fuel limit, call-frame support, or `gc-lang`/`ir-lang` wiring.
- The `serde` wire-format promise and the MSRV policy.

## Breaking changes

Two, both from the review, both trivial to migrate:

### `Chunk::emit` / `Chunk::patch` now speak `Addr`

`emit` returns `Addr` (`u32`) instead of `usize`, and `patch` takes `addr: Addr`. An emitted address is exactly a branch target, so it now feeds straight into a `Jump`/`patch` with no cast — the single most common builder pattern loses its `as u32` noise.

```rust
// before
let top = chunk.emit(Op::Le { dst: 4, lhs: 1, rhs: 2 });
chunk.emit(Op::Jump { target: top as u32 });

// after
let top = chunk.emit(Op::Le { dst: 4, lhs: 1, rhs: 2 });
chunk.emit(Op::Jump { target: top });
```

Migration: drop the `as u32` on back-patched targets. If you indexed `code()` with an emitted address, add `as usize` there instead.

### `Op::LoadConst` field `konst` renamed to `index`

The field that was named `konst` (to dodge the `const` keyword) is now `index` — a cleaner permanent name, and one that is also the `serde` wire name.

```rust
// before: Op::LoadConst { dst: 0, konst: k }
// after:
Op::LoadConst { dst: 0, index: k }
```

Migration: rename the field in `Op::LoadConst { .. }` literals and patterns.

### Also

`Vm::run` is now `#[must_use]` — discarding a successful run's `Value` is almost always a mistake. This is not itself breaking (you can still `let _ = vm.run(..)` to opt out).

## Performance

Unchanged from `0.2.5`. Local Criterion means (`cargo bench`, Windows x86_64, Rust stable), reusing one `Vm`:

- Straight-line expression (10 instructions): **~13 ns/run** — ~1.3 ns per instruction.
- Counted loop `sum 1..=1000` (~5 000 instructions executed): **~7 µs/run**.
- Counted loop `sum 1..=100000`: **~470 µs/run**.

Numbers are hardware-dependent; `cargo bench` reproduces them on your target.

## What's next

Additive `1.x` work, each non-breaking and independent of the frozen core:

- Function calls / multiple chunks (call frames, a call stack).
- An execution fuel / step limit for hard bounds on untrusted programs.
- `gc-lang` wiring for heap-allocated object values.
- `ir-lang` wiring: direct execution or lowering to `Chunk`.

## Installation

```toml
[dependencies]
bvm-lang = "1.0"

# persist compiled bytecode
bvm-lang = { version = "1.0", features = ["serde"] }
```

MSRV: Rust 1.85.
